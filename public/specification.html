1. プロジェクト概要

目的

管楽器リペアマン育成プロジェクト（repairman.gakkiou.com）のブログセクションに掲載する記事を、 AIを活用して自動生成し、WordPress上に下書き投稿するバックエンドシステムです。

解決する課題

  • 管楽器修理に関する専門的なブログ記事を、キーワード指定のみで高品質に生成
  • SEOを意識したタイトル・本文・構成の自動最適化
  • サムネイル画像の自動生成・アップロードにより、制作工数を大幅削減
  • WordPress REST APIを介した下書き自動投稿で、公開前の確認フローを維持

アプリケーション全体を束ねるルートモジュール。ConfigModule（グローバル）、WordpressModule、BlogGeneratorModule をインポートします。

  • GET / — ヘルスチェック用エンドポイント
  • 静的ファイル配信: public/ ディレクトリ

ブログ記事生成のコアモジュール。Claude APIによるコンテンツ生成、Imagen 3.0によるサムネイル生成、WordPress投稿をオーケストレーションします。

6. データフロー

ブログ記事生成の一連の処理フローです。

1
リクエスト受信

POST /blog-generator/generate にキーワード（必須）とトピック（任意）を送信

{
  "keywords": ["フルート", "タンポ交換", "やり方"],
  "topic": "フルートのタンポ交換手順と注意点"
}
2
Claude APIによる記事生成

キーワードとトピックを含むプロンプトをClaude Sonnet 4に送信。JSON形式（title, content, excerpt）で結果を受け取ります。

3
サムネイル画像生成

生成されたタイトルを基に、Google Imagen 3.0で管楽器修理をモチーフにしたサムネイル画像を生成。PNG形式のBufferを取得。

4
WordPress メディアアップロード

生成した画像をWordPress REST APIでメディアライブラリにアップロードし、メディアIDを取得。

5
WordPress 下書き投稿

タイトル・本文HTML・抜粋・アイキャッチ画像IDを指定して、status: 'draft'で下書き記事を作成。

6
レスポンス返却 
{
  "postId": 123,
  "title": "フルートのタンポ交換...",
  "excerpt": "記事の要約...",
  "status": "draft",
  "featuredMediaId": 456,
  "link": "https://repairman.gakkiou.com/?p=123"
}
画像生成に失敗した場合でも、記事の下書き作成はアイキャッチなしで続行されます（グレースフルデグラデーション）。

## ルール
- 読者は管楽器修理に関心のある人（修理技術者志望者、現役リペアマン、
  楽器演奏者、吹奏楽指導者）です
- 専門的な内容でありながら、初学者にもわかりやすい表現を心がけてください
- 修理の現場で実際に役立つ実践的な情報を含めてください
- 上記のキーワードで検索するユーザーが求めている情報を網羅してください
- キーワードを自然にタイトルと本文に含め、SEOを意識してください
- 具体的な手順、工具名、注意点など実務に即した内容を記載してください
- 1500〜2500文字程度で書いてください
- 見出し（h2, h3）を使って読みやすく構成してください
- 「現場で学び、現場で活きる」というサイトのコンセプトに沿ってください

## 出力フォーマット
以下のJSON形式で出力してください。JSON以外は出力しないでください。

{
  "title": "記事タイトル",
  "content": "<h2>見出し</h2><p>本文...</p>...",
  "excerpt": "記事の要約（100文字以内）"
}

プロンプト設計方針

  • ペルソナ設定: 管楽器修理の専門家として、現場経験に基づいた記事を生成
  • SEO最適化: 指定キーワードをタイトル・見出し・本文に自然に配置
  • 構造化: H2/H3見出しで論理的に構成し、WordPress上での可読性を確保
  • 文字数制御: 1500〜2500文字でSEOと可読性のバランスを維持
  • 出力形式: JSON（title, content, excerpt）で構造化し、パース容易に

10. WordPress連携

認証方式

  • 方式: Basic認証（Application Password）
  • ヘッダー: Authorization: Basic {base64(username:app_password)}
  • Base URL: {WORDPRESS_URL}/wp-json/wp/v2
WordPress管理画面の「ユーザー > アプリケーションパスワード」から生成したパスワードを使用します。 通常のログインパスワードではありません。

記事投稿フロー

ステップAPI概要
1POST /wp-json/wp/v2/mediaサムネイル画像をアップロード → メディアID取得
2POST /wp-json/wp/v2/posts記事を下書き投稿（title, content, excerpt, featured_media, status=draft）

Swellテーマ対応

対象サイトはWordPress Swellテーマを使用しています。生成するHTMLは以下に留意します。

  • Swellのブロックエディタ（Gutenberg）との互換性を持つ標準的なHTMLタグを使用
  • H2/H3の見出し構造はSwellの目次自動生成機能と連動
  • 記事のexcerptはSwellのブログカード表示に利用される
